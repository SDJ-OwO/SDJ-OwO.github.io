首先左側的圖，是源於新聞做的圖，從標題可得知，這是關於Covid-19確診病例的圖 ，我們針對其左下角的圖進行優化
資料來源 : 台灣疫情報告
新聞：成功預測本週確診下降　他再預告：6月5日變個位數

五月確診人數

 我認為原圖有著呈現過於簡單的問題，觀察者很難再看到的第一眼就能分出個資料的差異，且因為資料之間的差距太大，
前面幾天的資料難以看出到底有多少，也沒有各個長條所代表的數字。
 我的作法與原圖一樣是長條圖，但為了讓觀察者快速對資料分群，所以在其基礎之上多使用了顏色的標記，
讓觀察者能更快區分哪幾天的狀況是嚴重的、中等的、輕度的，這裡採用的顏色分群利用了上課所提到的colormaps，由於資料是
順序的而非發散的，所以我採用單色系的colormap且愈糟的情況(愈多確診案例)顏色的亮度愈暗。為了凸顯前面幾天較少人數的資料
儘管前幾天的資料與被背景都是較淡的顏色，但背景的亮度不強，所以能避免失焦的況狀發生。
 當觀察者將滑鼠移到長條上時，會在圖表的左方顯示當天的詳細資訊，讓觀察者進一步了講狀況，另外被指到的長條會變成深綠色
其他長條會變成薄荷色，長條和背景配色採用了Adobe Color Picker的補色，希望藉由這種配色方式將主角"長條"更加凸顯。






首先對於他對於新增染疫人口數的長條圖沒有進行太大的更改 ，我加上了一條折線圖是對於其總染疫人口數的圖，由於總人口跟新增人口 的座標軸是不一樣的，我在其右邊新增了座標軸，然後怕此折線圖影響到觀看長條圖 對長條圖放了當滑鼠移到長條上時，折線圖會變得較為透明；反之如此。

由於怕顏色過於相近，所以選擇了藍色跟紅色，因為此顏色相差較大的顏色。 而移到長條圖上把折線圖變透明的效果，並沒有完全透明，因為觀看長條圖時， 也可以跟折線圖一起觀看，而將圖轉為透明，則不會影響到觀看長條圖；反之如此


在原本的長條圖中，雖然可以看出每一次的確診人數， 卻很難看出疫情的趨勢變化，所以我就想藉由上色來看出相比前一天的浮動， 這一天的浮動是正向的OR反向的使用的公式為：
slope[0]=1
slope[n] = number[n]/number[n-1]
if (slope[n]>slope[n-1]) red
else green
我使用的顏色是依照前一天的斜率和今天的斜率相比，如果確診的情況變嚴重，就會是紅色，反之會是綠色， 從圖中真的可以看出疫情的趨勢，但老實說搭配折線圖可能會更好。
